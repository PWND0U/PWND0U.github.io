    Ubuntu下完美干净卸载docker

    docker docker ubuntu
    创建时间:2019-11-22 16:59
    阅读:
    1. 问题描述
    2. 解决途径
    3. 最后解决的方法

    问题描述

    • 因为想搭建的服务支持的docker版本与当前服务器中支持的不一致，需要切换版本
    • 需要干净卸载现有docker来支持新的服务

    解决途径

    1. 首先我在docker官方文档查阅

      • 官方给的解决方法：
      $ sudo apt-get purge docker-ce
$ sudo apt-get purge docker-ee
$ sudo rm -rf /var/lib/docker
    1. 按照操作之后发现并不能干净卸载docker，正要放弃之时，突然看到一篇帖子，说是和docker还有相关的残留包。

    最后解决的方法

    • 按照官方给的操作执行

      $ sudo apt-get purge docker-ce
$ sudo rm -rf /var/lib/docker
$ sudo rm -rf /var/run/docker

    • 查找残留包

      dpkg -l | grep docker

    • 删除相应的残留包

      sudo apt remove --purge 残留包名

    完美解决
